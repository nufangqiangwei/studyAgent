# 产品能力

本项目最终希望形成一个可学习、可扩展的 CLI agent 原型。它应该像开发者助手一样在终端里工作，但实现上保持简单清晰。

## 交互体验

基础形态是一个命令行程序：

```text
agent "帮我阅读这个项目并总结结构"
agent
> 修复当前测试失败的问题
```

交互层应该只负责输入输出、命令解析和展示，不直接包含 agent 推理、模型调用或工具执行逻辑。

## 核心能力拆解

### 1. 会话管理

- 创建一次用户任务会话。
- 保存用户输入、模型响应、工具结果。
- 支持多轮上下文。
- 控制最大上下文长度。

### 2. 上下文收集

- 读取当前工作目录。
- 搜索文件名和文本内容。
- 读取指定文件片段。
- 提取项目元信息，例如语言、测试命令、模块结构。

### 3. 模型调用

- 通过统一接口调用不同 LLM 供应商。
- 支持流式输出。
- 支持结构化响应。
- 支持模型配置和超时控制。

### 4. 工具调用

- 工具以注册表方式接入。
- 每个工具声明名称、参数、权限级别和执行函数。
- agent 内核只依赖工具接口，不依赖具体工具实现。
- 工具结果以结构化形式返回给模型和 UI。

### 5. 任务循环

典型循环如下：

1. 用户输入任务。
2. 系统收集必要上下文。
3. agent 调用模型生成下一步行动。
4. 如果需要工具，则执行工具并回传结果。
5. 如果需要修改代码，则生成补丁或写入文件。
6. 运行测试或检查命令。
7. 汇总结论并等待用户下一步。

### 6. 代码修改

- 优先使用补丁模型表达变更。
- 修改前应读取目标文件。
- 修改后应运行格式化、测试或静态检查。
- 对失败结果进行反馈和重试。

### 7. 安全与权限

- 区分只读工具和写入工具。
- 区分普通命令和高风险命令。
- 对删除、移动、网络请求等操作保留明确边界。
- 记录工具调用历史，便于调试和审计。

## 能力优先级

第一阶段优先实现：

- CLI 输入输出。
- 本地文件读取和搜索。
- 模型接口抽象。
- 简单工具注册和执行。
- 单轮任务处理。

后续再扩展：

- 多轮会话。
- 流式输出。
- 代码补丁。
- 测试反馈循环。
- 配置文件。
- 插件化工具。
