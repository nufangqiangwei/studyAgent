# 项目概览

本项目是一个用 Go 语言开发的 agent 学习项目，目标是理解并逐步实现开发者 CLI agent 的核心机制。

项目方向参考 Codex CLI、Gemini CLI、Claude Code 这类工具的使用方式，但重点不是复制某个产品，而是拆解和学习它们背后的通用能力：交互式命令行、上下文收集、模型调用、工具执行、代码修改、验证反馈和任务循环。

## 核心目标

- 构建一个可在终端中运行的 agent CLI。
- 支持用户输入自然语言任务。
- 能读取当前项目上下文，并将必要信息提供给模型。
- 能根据模型输出调用本地工具，例如读取文件、搜索文本、执行命令。
- 能生成或修改代码，并通过测试或命令验证结果。
- 保持 Go 模块之间解耦，便于替换模型、工具、存储和交互界面。

## 学习目标

- 理解 CLI agent 的基本任务循环。
- 学习 Go 项目结构和模块边界设计。
- 学习如何用接口隔离外部模型供应商。
- 学习工具注册、权限控制、执行结果回传等 agent 基础设施。
- 学习如何把复杂任务拆成可验证的小步骤。
- 学习如何设计可测试、可扩展的 agent 内核。

## 典型使用场景

- 让 agent 解释当前项目结构。
- 让 agent 搜索代码并定位问题。
- 让 agent 修改代码并运行测试。
- 让 agent 根据需求生成新模块。
- 让 agent 维护项目文档。

## 暂不追求的内容

- 不在早期实现复杂 IDE 集成。
- 不在早期实现多用户服务端。
- 不在早期实现复杂权限沙箱。
- 不把某个模型供应商绑定为唯一实现。
- 不为了功能堆叠牺牲模块边界。
